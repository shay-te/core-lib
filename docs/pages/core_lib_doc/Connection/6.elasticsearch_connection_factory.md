Elasticsearch clients manage connection pools internally and should be shared, not recreated per request. `ElasticSearchConnectionFactory` holds the client as a long-lived singleton and exposes it through the same `with conn.get() as client:` pattern as every other Core-Lib data source — so your `DataAccess` code looks identical regardless of backend. The `with` block does **not** close the connection on exit.

*core_lib.connection.elasticsearch_connection_factory.ElasticSearchConnectionFactory* [[source]](https://github.com/shay-te/core-lib/blob/master/core_lib/connection/elasticsearch_connection_factory.py){:target="_blank"}

## Configuration

`your_core_lib.yaml`
```yaml
# @package _global_
core_lib:
  data:
    elasticsearch:
      _target_: core_lib.connection.elasticsearch_connection_factory.ElasticSearchConnectionFactory
      config:
        url:
          protocol: https
          host: ${oc.env:ES_HOST}
          port: ${oc.env:ES_PORT}
          username: ${oc.env:ES_USER}
          password: ${oc.env:ES_PASSWORD}
```

`your_core_lib.py`
```python
from omegaconf import DictConfig

from core_lib.core_lib import CoreLib
from core_lib.helpers.config_instances import instantiate_config
from core_lib.connection.elasticsearch_connection_factory import ElasticSearchConnectionFactory


class YourCoreLib(CoreLib):
    def __init__(self, config: DictConfig):
        super().__init__()
        self.config = config
        es = instantiate_config(self.config.core_lib.data.elasticsearch, ElasticSearchConnectionFactory)
        self.search = SearchDataAccess(es)
```

`search_data_access.py`
```python
from core_lib.data_layers.data_access.data_access import DataAccess
from core_lib.connection.elasticsearch_connection_factory import ElasticSearchConnectionFactory


class SearchDataAccess(DataAccess):

    def __init__(self, db: ElasticSearchConnectionFactory):
        self._db = db

    def search(self, index: str, query: dict) -> dict:
        with self._db.get() as client:
            return client.search(index=index, body=query)

    def index(self, index: str, doc_id: str, document: dict):
        with self._db.get() as client:
            return client.index(index=index, id=doc_id, body=document)

    def delete(self, index: str, doc_id: str):
        with self._db.get() as client:
            return client.delete(index=index, id=doc_id)
```

# ElasticSearchConnectionFactory

## Functions

### \_\_init\_\_()

```python
def __init__(self, config: DictConfig):
```

**Arguments**

- **`config`** *`(DictConfig)`*: Elasticsearch section config. Must contain a `url` block with connection parameters (see YAML example above).

### client()

Returns the underlying `Elasticsearch` client instance directly, without going through the context manager.

```python
@property
def client(self) -> Elasticsearch:
```

**Returns**

*`(Elasticsearch)`*: The shared `elasticsearch.Elasticsearch` client.

### get()

Returns an `ElasticSearchConnection` context manager that yields the `Elasticsearch` client.

```python
def get(self, *args, **kwargs) -> ElasticSearchConnection:
```

**Returns**

*`(ElasticSearchConnection)`*: Context manager that yields the `Elasticsearch` client on `__enter__`. The connection is **not** closed on `__exit__` — the client is shared and long-lived.
