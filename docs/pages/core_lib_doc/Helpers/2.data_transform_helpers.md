Nested dict access in Python requires chained `obj['a']['b']['c']` calls with manual key-error handling. These helpers let you read and write any depth of nesting with a dot-path string, and return a safe default when the path is missing.

## Functions

### get_dict_attr()

*core_lib.data_transform.helpers.get_dict_attr()* [[source]](https://github.com/shay-te/core-lib/blob/master/core_lib/data_transform/helpers.py#L4){:target="_blank"}

Reads a value from a nested dict using a dot-separated path. Returns `default` if any key in the path is missing.

```python
def get_dict_attr(obj: dict, path: str, default=None):
```
**Arguments**

- **`obj`** *`(dict)`*: Dictionary to read.  
- **`path`** *`(str)`*: Dot-separated path to the value, e.g. `'education.school'`.
- **`default`** *`(any)`*: Default `None`. Returned when any key in the path is missing.

**Returns**

*`(any)`*: The value at the target path, or `default` if the path does not exist.

**Example**
```python
from core_lib.data_transform.helpers import get_dict_attr

data = {'name': 'Jon', 'education': {'school': 'Public School'}}

print(get_dict_attr(data, 'education.school'))   # Public School
print(get_dict_attr(data, 'name'))               # Jon
print(get_dict_attr(data, 'education.degree'))   # None
print(get_dict_attr(data, 'missing', 'unknown')) # unknown
```


### set_dict_attr()

*core_lib.data_transform.helpers.set_dict_attr()* [[source]](https://github.com/shay-te/core-lib/blob/master/core_lib/data_transform/helpers.py#L15){:target="_blank"}

Sets a value at a dot-separated path in a nested dict. Creates any missing intermediate keys automatically.

```python
def set_dict_attr(obj: dict, path: str, value) -> dict:
```
**Arguments**

- **`obj`** *`(dict)`*: Dictionary to update.  
- **`path`** *`(str)`*: Dot-separated path to write to.
- **`value`** *`(any)`*: The value to set.

**Returns**

*`(dict)`*: The same dict, mutated in place, with the value written at the given path.

**Example**

```python
from core_lib.data_transform.helpers import get_dict_attr, set_dict_attr

data = {'name': 'Jon', 'education': {'school': 'Public School'}}

set_dict_attr(data, 'education.school', 'International School')
print(get_dict_attr(data, 'education.school'))  # International School

new_data = set_dict_attr({}, '1.2.3', 'new dict')
print(new_data)  # {'1': {'2': {'3': 'new dict'}}}
```

### enum_to_dict()

*core_lib.data_transform.helpers.enum_to_dict()* [[source]](https://github.com/shay-te/core-lib/blob/master/core_lib/data_transform/helpers.py#L26){:target="_blank"}

Converts a Python `enum.Enum` class to a plain `dict` mapping member names to their integer values. Useful for serializing enum options to JSON or for frontend consumption.

```python
def enum_to_dict(enum: enum) -> dict:
```

**Arguments**

- **`enum`** *`(enum.Enum)`*: The enum class to convert.

**Returns**

*`(dict)`*: `{member_name: member_value}` for every member in the enum.

**Example**

```python
import enum
from core_lib.data_transform.helpers import enum_to_dict

class Status(enum.Enum):
    ACTIVE = 1
    INACTIVE = 0
    BANNED = 2

print(enum_to_dict(Status))  # {'ACTIVE': 1, 'INACTIVE': 0, 'BANNED': 2}
```
