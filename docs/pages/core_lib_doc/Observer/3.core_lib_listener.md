*core_lib.core_lib.CoreLibListener* [[source]](https://github.com/shay-te/core-lib/blob/master/core_lib/core_lib_listener.py#L7){:target="_blank"}

`CoreLibListener` is an abstract class that notifies you when your `CoreLib` instance starts up or shuts down — useful for starting background jobs, warming caches, or releasing resources cleanly.

## Usage

```python
from core_lib.core_lib_listener import CoreLibListener
from core_lib.core_lib import CoreLib

class YourCoreLib(CoreLib):
    def __init__(self, config: DictConfig):
        super().__init__()
        ...

class AppListener(CoreLibListener):

    def on_core_lib_ready(self):
        # called after start_core_lib() — start jobs, warm caches, etc.
        pass

    def on_core_lib_destroy(self):
        # called on __del__ — release resources, flush buffers, etc.
        pass

core_lib = YourCoreLib(config)
core_lib.attach_listener(AppListener())
core_lib.start_core_lib()
```

### update()

*core_lib.core_lib.CoreLibListener.update()* [[source]](https://github.com/shay-te/core-lib/blob/master/core_lib/core_lib_listener.py#L13){:target="_blank"}

Implemented from `ObserverListener` — routes `CORE_LIB_READY` and `CORE_LIB_DESTROY` events to the appropriate abstract method.

```python
def update(self, key: str, value):
```

**Arguments**

- **`key`** *`(str)`*: The lifecycle event key.
- **`value`**: Unused — always `None` for lifecycle events.

**Example**

```python
class Listener(CoreLibListener):

    def on_core_lib_ready(self):
        # Do some task here
        pass

    def on_core_lib_destroy(self):
        # Cleanups here
        pass

core_lib = YourCoreLib(config)
core_lib.attach_listener(Listener())
```
